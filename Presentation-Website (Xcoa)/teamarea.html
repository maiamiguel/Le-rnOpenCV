  • Week 6

M2 revision and feedback analysis

During this week, the team analysed the Milestone 2 feedback. Issues like scalability and complexity of the system raised by the evaluation team, were discussed in the team meetings and with the mentors. April
  • Week 7

OpenCV features and personal accounts

This week marked the initiation of the process to create personal accounts. One of the objectives of our web application is to guarantee the possibility of personalization for the user. To accomplish this the user may create and login to an account that will serve as repository for the user information and the manipulations done by him.
The progress with the OpenCV features is continuing so that the project can include a variety of different functions.

M3 and login process

The team analysed the topics suggested in the M3 documentations and defined roles for the members.The login process was implemented through the construction of an authentication module build to allow registration and login of users.It is still in the testing stage and needs to be completed with the implementation of a database for the users.

April
  • Week 9

M3 development and database definition

This week the team centered its development on the implementation of a database to aid the login process. This will be central point of storage for the user's account information. There were also some tweaks made to the frontend of the system with the creation of a registration page.
Milestone 3 is developing at a good pace and the team predicts that it will be ready for the Milestone deadline.

  • Week 10

Undo and redo features

One of the features that has been a major focus is the undo and redo of the manipulations made. The analysed the problem and defined a possible solution to be implemented. The choice was to use a support structure to save the manipulations so that they be evoked at any time. This will be subject to implementation in the next weeks and testing on the next stage.
