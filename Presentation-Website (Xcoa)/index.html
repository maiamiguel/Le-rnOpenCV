Le@rnOpenCV

Learning Visual Computing With OpenCV

OpenCV (Open Source Computer Vision Library) was initially developed by Intel in 2000. It is a multiplatform library built in C/C++ with support for Python, Java and Visual Basic. Its usage is free for educational purposes and is majorly used in app development in the areas of Visual Computing.
According to Intel, the most important developments to come in computer science will be in the visual aspects. Visually rich interfaces create new perspectives with multiple points of application. These can be used for simple entertainment, reproduction of virtual scenarios, identification of objects or people in real time or even allow the access on images of places that are inaccessible, for example in Medicine.
Our project is an attempt to facilitate the interaction and manipulation of images through a friendly user web service. It aims to permit the user an access to OpenCV functionalities without the need of programming knowledge.






Features

Since our image processing solution is based on a web service, the features that we pretend to implement are based on high interactivity between service and client. There will be features related to image processing itself and features related to the user's data.

Learn image processing

Learn and study image processing and visual computing algorithms using the OpenCV library.

Apply image filters

Use a variety of interactive and easy-to-use effects to your images and visualize the results quickly.

Create customized scripts

Create scripts with the image effects that you want and use them on any image.

Add OpenCV functions

Add OpenCV functions beyond those that already exist on our application, thus increasing the system's expandability.

Access the code and OpenCV information

Access the code that generated the image transformation, with a link to further information about the OpenCV function used behind.

Create a personal account

Create a personal account that stores your favorite images (with all the effects applied) and your custom scripts and OpenCV functions.

Schedule of features / functionalities release vs milestones

  • Milestone 2

    User - server communication will be implemented by establishing the server technology to transfer information.
    First version of initial web interface will be defined and simple OpenCV functions will be available.

  • Milestone 3

    The initial simple button submission will be replaced by a more intuitive and simplistic drag-n-drop style submission.

  • Milestone 4

    A more complete set of OpenCV functions will be available to the user with different levels of complexity and effect combinations.
    A gallery of the recent interaction will be available to permit the back-roll to a previous state or the usage of the previous applied effects.

  • Milestone 5

    The OpenCV information that allows the creation of the image effects and the corresponding scripts will be shown to user, with the orientation for user that want a deeper understanding of the functions.
    Users will be able to expand the OpenCV functions available by adding new functions.

  • Milestone 6

    At this stage the user will be given the possibility to create an account which will allow him to store information relative to all the interaction with our application, namely the changes made to past images and all the commands and combination of commands executed before and all the customized scripts and functions.

  • Client side

    The user must access the project page through a HTML and CSS compatible browser. The communication between the page and the servers is done through the use of a Javascript library called jQuery.

  • Server side

    The backend services will run on applications using Python. The web server will be running on the CherryPy framework, which allows a simplistic interaction between the backend and frontend. All the image processing and manipulation will be done by the OpenCV library that provides a great variety of functions available for the user. To save user data and information a database server may be added later to enable features in the future. The backend services may run on a single machine, based on a virtual or physical environment.

Risks and Issues

Server Unavailability

Our project uses several platforms which require an internet connection. Due to this fact, some risks associated with server unavailability due to internet breakdown

Package loss

The connection through the Internet creates a great number of packet exchanges that might some conflicts and collision leading to packet loss

Incompatibilities

The users interface must be compatible with the technologies on which the web page is developed on, namely HTML, CSS and JavaScript libraries

Client data management

Data regarding the user will be stored and must be securely managed to offer through a simplistic and effectfull way
