Developer Area

HOW TO DEPLOY OUR SYSTEM?

The deployment of our system needs only a simple web browser with support for JavaScript and an Internet connection. The system is based on a remote execution of the operations making it possible for anyone to use it without the need of any kind of special equipment or installation.

The system uses:

  • Cherrypy framework
  • OpenCV library
  • SQLite3

The user interface is a web application portal that uses HTML, CSS and JavaScript integrated with jquery request handling. The simplistic implementation allows the user to have a better and more fulfilling experience.

The back-end of our system is python (python 2.7) based and uses Cherrypy as the HTTP framework allowing the developer to build interactive web applications. The choice of Cherrypy was based on its easy to use and simplistic implementation for the tasks at hand. It is responsible for the connection between the system’s frontend and the server side where the modules operate on the images.

The choice for the manipulation module was made by analysing the advantages of different types of tools for the task. The choice was made to use OpenCV library. Its bindings with python contributed to the choice and as Computer Vision learning tool it is excellent, with a wide variety of real world applications.

For the database we chose SQLite3 because of it is simple to use, configure and manage. Due to it being easily integrated into the system build it can run on the same machine, making it faster and more reliable.

HOW TO RUN OUR SYSTEM?

At this point, the system can be run on a local server, but the objective is to establish it on an online server so that it can be accessed everywhere through the use of a browser. To run the application it is necessary that the machine satisfies some requirements, namely the installation of the Cherrypy framework and the OpenCV library for integration with python.

CherryPy can be easily installed via common Python package managers such as setuptools or pip:

  • $ easy_install cherrypy
  • $ pip install cherrypy

The OpenCV installation needs firstly to download some dependencies to work properly. It relies on dependencies like cmake to automatically configure our OpenCV build.. It can be installed by using:

  • $ sudo apt-get install build-essential cmake pkg-config

In order to load the images from disk, OpenCV calls other image I/O libraries that actually facilitate the loading and decoding process:

  • $ sudo apt-get install libjpeg8-dev libtiff5-dev libjasper-dev libpng12-dev

To process video other packages are needed:

  • $ sudo apt-get install libavcodec-dev libavformat-dev libswscale-dev libv4l-dev
  • $ sudo apt-get install libxvidcore-dev libx264-dev

Internally, the module that handles OpenCV GUI operations is highgui and it relies on the GTK library:

  • $ sudo apt-get install libgtk-3-dev

To end the dependencies section for the OpenCV installation it is necessary to install Python development headers and libraries for both Python 2.7:

  • $ sudo apt-get install python2.7-dev

For the final installation of OpenCV module with python integration just need to run:

  • $ pip install opencv-python

If necessary the SQLite database can be installed through the download and installation of packages:

  • $ sudo apt-get install sqlite3 libsqlite3-dev

After all the installation of all the packages necessary the application code can be run through the terminal interface by issuing the command:

  • $ python server.py

This will setup the application for use, and making it possible for the user to interact with the system.

Check Instalation

After the instalation the user should be able to check his verion of Python and OpenCV, it should be 2.7 for Python and 3.1 for OpenCV or latest.

  • Check Python version:
    • python -V

  • Check OpenCV version:
    $ python
    >>> import cv2
    >>> cv2.__version__

    • HOW TO DEVELOP ON OUR SYSTEM?

    The architecture consist of a web application, which offers the user several manipulation options through the access in a menu to the OpenCV functions. These functions are open source and are available for anyone to use. Therefore any interested party could use our system and develop upon it by adding new type of resources and functionalities. The independence of the OpenCV module allows changes to be integrated easily.

    DESIGN AND IMPLEMENTATION OPTIONS

    Regarding the model, the exchanges made from the front-end to the back-end modules were all made through POST and GET messages generated with Asynchronous JavaScript And XML (AJAX) which requires the TCP/IP network protocol. GET messages were used mostly when it was required to display a certain image to the user while POST messages were used mostly whenever a user applied a new function, submited a new script, saved a script or a custom function.

    For the web framework we chose to use CherryPy due to its rapid deployment, minimalistic nature and simple implementation. As for the database engine we chose SQLite due to its bindings with the python programming language (which was used to build the server module (CherryPy), the script module and the OpenCV library image processing algorithms) and because of the features and functionalities it offers in web application software for local/client storage.
