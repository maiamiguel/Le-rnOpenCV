Specifications

SCENARIOS WHERE THE SYSTEM IS APPLIED

  • Learning openCV functionalities and algorithms
  • Testing and comparing openCV funcionalities and algorithms
  • User self-learning process
  • Conducting an workshop

ACTORS - WHO USES THE SYSTEM

  • People that don't have a programming background and want to interact and learn image processing
  • Image processing and openCV enthusiasts

SOFTWARE REQUIREMENTS

Operating Environment

  • OE-1: The system must be suported in every web browser
  • OE-2: The system must operate continuously on the server
  • OE-3: The system must use a database to store all the users' information
  • OE-4: The server must run the latest versions of OpenCV and CherryPy libraries

Design And Implementation

  • C0-1: The system must store and keep all the user’s information in SQLite database
  • C0-2: All HTML code must be in accordance with HTML5 Standard

Assumptions and Dependencies

  • Assumptions
    • AS-1: The services provided by the platform must be available 24/7
    • AS-2: The selected image must be always on the workspace in order to be edited
    • AS-3: The request to apply an effect to an image must be correctly processed by the server
    • AS-4: After applying an effect to an image, the user must receive the new image on his workspace, processed by the server and with the respective changes
  • Dependencies
    • DE-1: The good functioning of the application, i.e. the response time of the server, depends on the amount of traffic in the network as well as the number of requests to be processed by the server

SYSTEM FEATURES

  • FE-1: Select one or more images by browsing on the user's computer or by drag-n-dropping them on the application workspace
  • FE-2: Apply a set of effects and filters to one or more images
  • FE-3: Adjust the parameters of the OpenCV function to be applied on the image, if applicable
  • FE-4: Download the edited image to the user's computer
  • FE-5: Go back in the image changes (undo)
  • FE-6: Go forward in the image changes (redo)
  • FE-7: Visualize the edited images by selecting one of them from the image gallery
  • FE-8: Create a python script with several effects, which will be available for the user to use it on future image editing
  • FE-9: Sharing images on social networks
  • FE-10: View a brief description of each available OpenCV function (meaning, parameters, etc)
  • FE-11: View the complete documentation of the OpenCV function through the redirect to the official OpenCV library website
  • FE-12: Save the image and its changes to the server
  • FE-13: Create a personal account
  • FE-14: Provide a tutorial/user manual to guide the user through the platform

EXTERNAL INTERFACE REQUIREMENTS

User Interface

  • UI-1: The application must provide a brief tutorial explaining its operations
  • UI-2: The system must provide an error message if the image has not been loaded (through import or drag-n-drop) correctly on the workspace
  • UI-3: The system must provide an error message if the image has not been loaded (by the server) correctly on the workspace
  • UI-4: The system must provide an error message in case of login failure
  • UI-5: The system must provide an error message if the effect chosen by the user has not been applied correctly

Software Interface

  • SI-1: Authentication System
  • SI-1.1: The authentication system must allow the registration and authentication of a user
  • SI-2: User Management System
  • SI-2.1: The system must list the most recent changes to the image currently being edited (image gallery)
  • SI-2.2: The system must list the images associated with the user in the user library
  • SI-3: Image Management System
  • SI-3.1: The system must allow effects and filters to be applied to a selected image
  • SI-3.2: The system must allow you to rewind in the process of changing the image
  • SI-3.3: The system must allow you to resume the changes made to an image

Communication Interface

  • CI-1: The authentication system, after registration of a user in the platform, must send an informative email to the user containing his personal information regarding the application (username, password)
  • CI-2: When a user changes his password, the authentication system must send an email to the user informing him of this change

QUALITY ATTRIBUTES

Usability Requirements

  • USE-1: 95% of the requested changes to the server must be submitted successfully

Security Requirements

  • SEC-1: All information about users must be protected

Availability Requirements

  • AVA-1: The system must be available at least 95% during the day between 10h00 and 00h00 and 50% between 00h00 and 10h00

Robustness Requirements

  • ROB-1: If a connection breaks during the editing of an image, the system must save the current image. Once the user restarts the service, the edited image must be available on the user's workspace

SYSTEM USE CASES

  • UC-1: Select an image
  • UC-2: Apply effect to an image
  • UC-3: Download an edited image
  • UC-4: Undo/Redo image changes
  • UC-5: Visualize the image gallery
  • UC-6: Create a custom script
  • UC-7: Visualize the generated python code
  • UC-8: Visualize the OpenCV function documentation
  • UC-9: Add image to favorites

USE CASE DIAGRAM


INDIVIDUAL USE CASE SPECIFICATION (MOST RELEVANT)

ID and Name UC-2: Apply an effect to an image
Description The registered/unregistered user selects the desired effect to apply to the image. Depending on the selected effect, the registered/unregistered user may also need to adjust certain parameters for the function associated to that effect.
Actors Registered/Unregistered user
Frequency of use Available 24 hours per day and 7 days per week.
Main course 1. The registered/unregistered user selects the desired effect to apply to the image, adjusting certain parameters if applicable.
Alternative course --
Priority High
Observations This use case can only be performed if there is at least 1 image ready to be edited on the workspace.


ID and Name UC-5: Visualize the image gallery
Description The registered/unregistered user visualizes the edited images by clicking on the thumbnails of the image gallery.
Actors Registered/Unregistered user
Frequency of use Available 24 hours per day and 7 days per week.
Main course 1. The registered/unregistered user clicks on the thumbnails of the image gallery and the edited images is displayed on the workspace.
Alternative course Step 1. The registered/unregistered user does not have edited images in the image gallery. The registered/unregistered user should select an image and apply effects in order to advance in the main course steps.
Priority Medium
Observations This use case can only be performed if there is at least 1 edited image in the image gallery.


ID and Name UC-6: Create a custom script
Description The registered user creates a custom script with several effects that can be used sequentially on any image.
Actors Registered user
Frequency of use Available 24 hours per day and 7 days per week.
Main course 1. The registered user inputs his access credentials on the platform;
2. The registered user selects an image;
3. The registered user applies effects to the image;
4. The registered user clicks on the "plus" button inside the script box.
Alternative course --
Priority Medium
Observations This use case can only be performed if the registered user has applied at least 1 effect to an image in order to create a custom script.


ID and Name UC-7: Visualize the generated python code
Description The registered user uses the terminal to apply an effect to an image and visualizes the generated python code of the image transformation.
Actors Registered user
Frequency of use Available 24 hours per day and 7 days per week.
Main course 1. The registered user inputs his access credentials on the platform;
2. The registered user selects an image;
3. The registered user applies an effect to the image using the terminal.
Alternative course --
Priority Medium
Observations --


ID and Name UC-9: Add image to favorites
Description The registered user adds an image from the image gallery to favorites.
Actors Registered user
Frequency of use Available 24 hours per day and 7 days per week.
Main course 1. The registered user inputs his access credentials on the platform;
2. The registered user applies effects to an image;
3. The registered user clicks on the star button of the image.
Alternative course --
Priority Medium
Observations This use case can only be performed if there is at least 1 edited image in the image gallery.



USER STORIES (MOST RELEVANT)

  • US-1: Selection of an image to edit
  • US-2: Application of an image effect
  • US-3: Creation of a custom script
  • US-4: Visualization of the generated python code
  • US-5: Addition of an image to favorites

ID and Name US-1: Selection of an image to edit
Associated use case UC-1
Description As a registered/unregistered user
I want to be able to select an image from my computer
So that I can edit it by applying openCV functions.
Acceptance criteria Given a registered/unregistered user of the platform
When he selects an image from his computer
Then the image is displayed on the workspace and is ready to be edited.


ID and Name US-2: Application of an image effect
Associated use case UC-2
Description As a registered/unregistered user
I want to be able to apply an image effect
So that I can analyse the image with that effect.
Acceptance criteria Given a registered/unregistered user of the platform
When he has an image on his workspace
And he selects an effect
And specifies the required parameters
Then the selected effect is applied to the image.


ID and Name US-3: Creation of a custom script
Associated use case UC-6
Description As a registered user
I want to create a new custom script
So that I can combine several effects and use them on future image editing.
Acceptance criteria Given a registered user of the platform
When he clicks on the “plus” button inside the script box
Then he stores all the effects in a single script
And he can use it on any image.


ID and Name US-4: Visualization of the generated python code
Associated use case UC-7
Description As a registered user
I want to visualize the generated python code
So that I can learn the python code syntax.
Acceptance criteria Given a registered user of the platform
When he has an image on his workspace
And he applies an effect to the image from the terminal
Then the generated python code is shown in the terminal.


ID and Name US-5: Addition of an image to favorites
Associated use case UC-9
Description As a registered user
I want to add an image to favorites
So that I can visualize the effects that were applied and download the script of that image.
Acceptance criteria Given a registered user of the platform
When he has an image in his image gallery
And he clicks on the star button of that image
Then the image is saved to his favorites page with the applied effects and download script link.
